# Creative Test Agent — Обзор продукта

## Что это такое
Creative Test Agent — это локальная, работающая offline (closed-loop) система для автоматического pre-testing креативных материалов (рекламных текстов, изображений, видео) на соответствие брендбуку, tone of voice, рекламному законодательству и целевой аудитории перед запуском.

## Принцип работы
- Все данные и LLM работают локально — никакие данные не покидают контур организации.
- Нет облачных LLM (OpenAI, Anthropic, Gemini, Perplexity), внешних API или облачного хранилища.
- Система использует локальный LLM-рантайм (Ollama, LM Studio или совместимые) через единый абстрактный слой.

## Основные модули
- **Управление клиентами и проектами** — группировка креативов, брендбуков и тестов по кампаниям.
- **Брендбуки** — загрузка и ingestion фирменных руководств, правил и compliance-требований.
- **Креативы** — загрузка текстовых и файловых креативов с автоматическим извлечением текста.
- **Тестирование** — запуск структурированных тестов по рубрикам с оценками и рекомендациями.
- **Batch-тестирование** — массовый запуск тестов нескольких вариантов креативов.
- **Отчёты** — генерация детальных отчётов, A/B сравнение, экспорт в DOCX/PPTX/PDF-ready.
- **Рецензирование** — human-in-the-loop review с решениями (approve/changes/reject).
- **База знаний** — накопление инсайтов, feedback и уроков для будущих кампаний.
- **Оценка качества** — eval-кейсы для проверки качества ответов LLM.
- **Аудит** — логирование всех событий и prompt trace для отладки.

## Роли пользователей
- **Viewer** — только просмотр данных.
- **Analyst** — создание и запуск тестов, генерация отчётов.
- **Manager** — управление проектами, клиентами, пользователями.
- **Admin** — полный доступ, управление системой.

## Рабочий процесс
1. Загрузить demo-профиль или создать клиента/проект.
2. Загрузить креативы и брендбук.
3. Настроить рубрики тестирования.
4. Запустить batch-тест кампании.
5. Просмотреть сводку и сравнение вариантов.
6. Экспортировать отчёт.
7. Создать рецензию.
8. Сохранить feedback в базу знаний.

## Режимы развёртывания
- **Локальный (dev)** — `make run`, SQLite, stub LLM.
- **Серверный** — Docker Compose с SQLite/Postgres, локальный LLM.

## Ограничения
- Не облачный SaaS, не multi-tenant.
- Нет встроенного биллинга.
- Нет Kubernetes-оркестрации.
- Видео-анализ только через студийный импорт (не real-time).
