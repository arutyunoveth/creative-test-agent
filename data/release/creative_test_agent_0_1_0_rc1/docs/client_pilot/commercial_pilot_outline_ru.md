# Коммерческий пилот

## Цель

Предоставить агентству инструмент для предварительного тестирования креативов в локальном closed-loop режиме, оценить эффективность и принять решение о полноценном внедрении.

## Продолжительность

Ориентировочно **X–Y рабочих дней** после согласования требований и предоставления материалов.

Этапы:
1. Онбординг и настройка — 1–2 дня.
2. Тестирование креативов — 3–5 дней.
3. Анализ результатов и обратная связь — 1–2 дня.

## Этапы

| Этап | Описание | Длительность |
|------|----------|--------------|
| 1. Подготовка | Согласование scope, загрузка профиля бренда, аудиторий, рубрики. | 1–2 дня |
| 2. Тестирование | Загрузка креативов, запуск тестов, генерация отчётов, A/B сравнение. | 3–5 дней |
| 3. Анализ | Разбор отчётов, корректировка критериев, повторное тестирование. | 1–2 дня |
| 4. Подведение итогов | Демонстрация результатов, сбор обратной связи, решение о следующих шагах. | 1 день |

## Результаты

По итогам пилота предоставляются:

- Настроенная локальная инстанция системы с профилем клиента.
- Протестированные креативы с отчётами по каждому.
- A/B сравнение вариантов.
- Рекомендации по улучшению креативов.
- Заключение о применимости инструмента для задач агентства.
- Предложение по дальнейшему сотрудничеству.

## Что нужно от клиента

- Бренд-профиль: tone of voice, restrictions, claims policy.
- Профили целевой аудитории (до 5 сегментов).
- Рубрика оценки (можно использовать стандартную).
- Креативы для тестирования (тексты, PDF, изображения).
- Обратная связь по формату отчёта и результатам.

Для пилота достаточно 3–5 креативов (или 2–3 варианта одного сообщения).

## Что поставляется по итогам

- Документация по установке и настройке.
- Pilot profile в JSON-формате (возможность повторной загрузки).
- Все отчёты в Markdown и HTML.
- Client pilot pack (документация, скрипты, профиль).

## Возможные дальнейшие доработки

- Подключение локальной LLM (Ollama, LM Studio) для более глубокого AI-анализа.
- Миграция на PostgreSQL.
- Интеграция с внешними системами.
- Расширение панели критериев.
- Поддержка дополнительных форматов файлов.
- Разработка пользовательского интерфейса под бренд клиента.

## Риски и ограничения

| Риск | Описание | Митигация |
|------|----------|-----------|
| Ограничения stub-режима | В режиме по умолчанию не используется реальный AI-анализ | Подключение локальной LLM на следующем этапе |
| Производительность | SQLite может быть узким местом при высокой нагрузке | Миграция на PostgreSQL при необходимости |
| Требования к оборудованию | Локальная LLM требует ресурсов (RAM, GPU) | Stub-режим работает на любом оборудовании |
| Безопасность данных | MVP-версия не имеет разграничения прав | Пилот проводится на доверенной машине |

## Стоимость

Стоимость: **обсуждается после уточнения объёма пилота.**

Факторы, влияющие на стоимость:
- Количество тестируемых креативов.
- Необходимость подключения локальной LLM.
- Интеграция с внешними системами.
- Кастомизация отчётов и критериев.
- Срочность и объём поддержки.

## Юридические условия

Обсуждаются отдельно. Рекомендуется включить в договор:
- Условия обработки данных (локально, без передачи третьим сторонам).
- Отсутствие гарантий на результаты AI-анализа.
- Интеллектуальная собственность на креативы остаётся у клиента.
- Конфиденциальность материалов пилота.
