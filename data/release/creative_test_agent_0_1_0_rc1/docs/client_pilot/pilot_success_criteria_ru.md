# Критерии успешности пилота

## Технические критерии

- Система запускается локально одной командой (`make run` или `uvicorn src.main:app --reload`).
- `make pilot-check` проходит без ошибок.
- `make check-closed-loop` подтверждает закрытый контур.
- Креативы загружаются (txt, md, pdf, png, jpg, webp).
- Тестовые прогоны завершаются со статусом `completed`.
- Отчёты формируются в Markdown и HTML.
- A/B сравнение вариантов работает и возвращает осмысленные результаты.

## Бизнес-критерии

- Команда агентства понимает структуру и выводы отчёта без дополнительных пояснений.
- Отчёт помогает улучшить креатив до показа клиенту.
- Рискованный креатив с преувеличениями получает предупреждения (brand safety).
- Сравнение вариантов помогает выбрать лучший или выявить слабые места.
- Формат отчёта пригоден для внутренней работы агентства.
- Время от загрузки креатива до получения отчёта не превышает 1–2 минут.

## Критерии для клиентского пилота

- Клиент подтвердил, что понимает концепцию closed-loop и локальной обработки.
- Клиент согласовал пилотный скоуп (какие креативы, сколько вариантов, какие критерии).
- Клиент получил демонстрацию системы (live или запись).
- Клиент дал обратную связь по формату отчёта и usability.
- Согласованы дальнейшие шаги: расширение пилота, подключение LLM, коммерческое внедрение.

## Ограничения интерпретации

- **Синтетическая аудитория не заменяет реальные фокус-группы.** Система симулирует реакции на основе профилей, но не может предсказать реальное поведение людей.
- **Stub-режим не использует AI.** В режиме по умолчанию все оценки и риски основаны на детерминированных правилах. Для более глубокого анализа рекомендуется подключить локальную LLM.
- **Выводы требуют проверки маркетологом.** Система — инструмент поддержки принятия решений, а не замена экспертной оценки.
- **Итоговые решения остаются за командой агентства.** Отчёты и рекомендации носят справочный характер.

## Метрики пилота

| Метрика | Целевое значение |
|---------|-----------------|
| Тестовые прогоны успешны | 100% |
| Отчёты содержат все разделы | 100% |
| Brand safety обнаруживает явные нарушения | Да |
| Клиент подтверждает пользу инструмента | Да |
| Система не использует облачные сервисы | Подтверждено `check-closed-loop` |
