# Технический обзор

## Общая схема

```
Креатив (txt, pdf, png, jpg, webp)
  → Локальный парсинг (текст, метаданные)
  → Структурированный workflow (6 этапов)
    → Creative Summary
    → Audience Simulation
    → Brand Safety Review
    → Brandbook Compliance Review
    → Rubric Scoring
    → Final Recommendation
  → Локальный LLM/Vision provider (stub / Ollama / LM Studio)
  → Результаты (scorecard, риски, рекомендации)
  → Генерация отчёта (Markdown / HTML / DOCX / PPTX / PDF-ready)
  → A/B сравнение вариантов
  → Экспорт отчётов: Word, PowerPoint, PDF-ready HTML
```

Все данные остаются локально. Ни один этап не требует интернет-соединения.

## Backend

Стек:
- **Python 3.14+** — язык разработки.
- **FastAPI** — веб-фреймворк для API.
- **SQLAlchemy** — ORM для работы с базой данных.
- **SQLite** — база данных по умолчанию.
- **Pydantic** — валидация данных и схемы.
- **Jinja2** — шаблонизатор для веб-интерфейса.

Backend реализован как монолитное FastAPI приложение с модульной структурой:

```
src/
  main.py                    — точка входа, подключение роутов
  shared/                    — общие компоненты (config, db, errors)
  modules/                   — функциональные модули
    brand_profiles/          — управление брендами
    audience_profiles/       — управление аудиториями
    creative_assets/         — загрузка и управление креативами
    test_rubrics/            — критерии оценки
    test_runs/               — запуск тестов (pipeline)
    report_generator/        — генерация отчётов
    audit_log/               — логирование событий
    brandbooks/              — брендбуки (загрузка, чанкование)
    knowledge_base/          — база знаний (поиск, контекст)
    ui/                      — веб-интерфейс
```

## Локальные LLM-провайдеры

| Провайдер | Описание | Требования |
|-----------|----------|------------|
| **Stub** (по умолчанию) | Детерминированные ответы по правилам | Нет |
| **Ollama** | Локальный запуск LLM через Ollama | Установленная Ollama, скачанная модель |
| **LM Studio** | Локальный запуск через LM Studio | Установленная LM Studio, загруженная модель |

Настройка через переменные окружения:
- `CTA_LLM_PROVIDER=stub|ollama|lmstudio`
- `CTA_LLM_MODEL` — название модели
- `CTA_LLM_BASE_URL` — адрес локального сервера

## Vision/OCR слой

| Провайдер | Описание | Требования |
|-----------|----------|------------|
| **Stub** (по умолчанию) | Детерминированные ответы | Нет |
| **Local OCR** | Распознавание текста Tesseract | Tesseract OCR + pytesseract |
| **Local VLM** | Визуальное описание через Ollama | Ollama + vision model |
| **Hybrid** | OCR + VLM вместе | OCR и VLM настроены |

## Хранение данных

- **Файлы**: локальная файловая система (директория `data/storage/`).
- **База данных**: SQLite (`creative_test_agent.db`), всё в одном файле.
- **Метаданные**: JSON-поля в SQLite для гибкости схемы.

## Отчёты

- **Markdown**: полный текст отчёта с таблицами и разделами.
- **HTML**: стилизованная веб-версия для просмотра.
- **JSON**: структурированные данные через API.
- **A/B сравнение**: дельты оценок и сводка по вариантам.

## UI

Веб-интерфейс доступен по адресу `http://localhost:8000/`. Включает:
- Dashboard со статусом системы.
- Управление креативами, брендами, аудиториями, рубриками.
- Запуск тестов и просмотр отчётов.
- Сравнение вариантов.

## Ограничения текущей версии

- SQLite: не предназначен для высоких конкурентных нагрузок.
- Stub-режим: не использует реальный AI-анализ.
- Локальная LLM/Vision: опциональна и требует дополнительной настройки.
- Нет разграничения прав пользователей.
- Нет API для внешней интеграции.
- Видео и аудио не поддерживаются.
