# Сценарий клиентской демонстрации

## Вводная (1 минута)

"Мы разработали инструмент для предварительного тестирования креативов — до того, как они попадают в производство или показываются клиенту. Всё работает локально, без передачи данных в облачные сервисы."

## Экран 1: Дашборд (30 секунд)

Покажите главную страницу. Обратите внимание на:

- "Closed-loop" статус — зелёная метка, означающая что все данные обрабатываются локально.
- Статус базы данных — зелёный.
- "Всё, что вы видите, работает на вашем компьютере. Никакие креативы, бренд-данные или результаты тестов не отправляются вовне."

## Экран 2: Загрузка креатива (1 минута)

1. Перейдите в "Creative Assets".
2. Покажите, что можно загрузить текстовый файл, PDF или изображение.
3. Загрузите пример: "Давайте возьмём три варианта одного сообщения."

Покажите три креатива NovaBank:
- Вариант A (Practical) — чёткий, прагматичный.
- Вариант B (Freedom) — эмоциональный, вдохновляющий.
- Вариант C (Risky) — содержит преувеличения.

"Обратите внимание: мы специально добавили рискованный вариант, чтобы проверить, как система реагирует на сомнительные формулировки."

## Экран 3: Запуск теста (1 минута)

1. Создайте тест для варианта A.
2. Нажмите "Run Analysis".
3. Покажите, что тест завершился со статусом "completed".

"За пару секунд система проанализировала креатив по нескольким критериям: чёткость сообщения, запоминаемость, соответствие аудитории, призыв к действию, доверие, соответствие бренду, риск негативного восприятия, уникальность."

## Экран 4: Отчёт (1 минута)

Откройте отчёт для варианта C (Risky).

Покажите:
- **Executive Summary** — краткое резюме.
- **Scorecard** — таблица с оценками по каждому критерию.
- **Brand Safety and Risks** — раздел с рисками (здесь должны быть HIGH риски для варианта C).

"Система автоматически обнаружила проблемные формулировки: гарантии, отсутствие рисков, абсолютные утверждения. Всё это противоречит политике бренда."

## Экран 5: Сравнение вариантов (1 минута)

1. Перейдите на страницу сравнения.
2. Выберите варианты A, B и C.
3. Покажите результат сравнения.

"Система не только оценивает каждый креатив по отдельности, но и сравнивает их между собой. Это помогает выбрать лучший вариант и понять, какие именно аспекты нужно доработать."

## Экран 6: Демонстрация closed-loop (30 секунд)

Вернитесь на дашборд.

"Хочу подчеркнуть важный момент: все данные остаются на вашем компьютере. Мы не используем OpenAI, Google Cloud или любые другие облачные сервисы. Это особенно важно, если вы работаете с конфиденциальными креативами до их официального релиза."

Дополнительно можно показать:
- `GET /vision/health` — показывает, что используется stub-режим (zero cloud).
- `GET /llm/health` — показывает, что LLM-провайдер — stub.

## Вопросы к клиенту после демо (30 секунд)

1. "Какие типы креативов вы бы хотели тестировать в первую очередь?"
2. "Какие критерии оценки最重要 для вашего бренда?"
3. "Есть ли у вас требования по конфиденциальности данных, которые мы должны учесть?"
4. "Хотели бы вы попробовать интеграцию с локальной LLM (Ollama) для более глубокого анализа?"
5. "Какие форматы отчётов вам нужны для согласования с клиентом?"

## Технические заметки для демонстратора

- Для демо не требуется интернет.
- Для демо не требуется GPU.
- Для демо не требуется установка Tesseract или Ollama.
- Режим stub гарантирует, что всё будет работать даже на слабом ноутбуке.
- При необходимости можно показать реальную LLM (Ollama), если она установлена, но это опционально.
