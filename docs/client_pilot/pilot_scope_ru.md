# Рамка пилота Creative Test Agent

## Цель пилота

Проверить, как предварительное тестирование креативов с помощью локального AI-инструмента может повысить качество материалов до показа клиенту и сократить количество итераций согласования.

## Что входит в пилот

- Загрузка текстовых креативов, PDF-файлов и изображений для анализа.
- Автоматическое структурированное тестирование по 8 критериям: чёткость сообщения, запоминаемость, соответствие аудитории, призыв к действию, доверие, соответствие бренду, риск негативного восприятия, уникальность.
- Симуляция реакции целевой аудитории (синтетические профили).
- Проверка brand safety: выявление рискованных формулировок (преувеличения, гарантии, zero-risk language).
- Детальный отчёт в Markdown и HTML с Executive Summary, Scorecard, рисками и рекомендациями.
- A/B сравнение нескольких вариантов креатива.
- Визуальный анализ изображений (в stub-режиме — без GPU/OCR).
- Веб-интерфейс для управления креативами, запуска тестов и просмотра отчётов.

## Что не входит в пилот

- Полноценные маркетинговые исследования или фокус-группы.
- Замена социологических опросов и real-user testing.
- Автоматическое создание креативов с нуля (генерация контента).
- Интеграция с внешними системами (CMS, CRM, ad-платформами).
- Многотенантность или разграничение прав доступа.
- Облачные LLM-провайдеры (OpenAI, Anthropic, Google Gemini).
- Облачные OCR/Vision API (Google Vision, AWS Textract, Azure OCR).
- Обработка видео или аудио.

## Какие материалы можно тестировать

- Текстовые креативы (копирайтинг, слоганы, описания).
- PDF-документы (презентации, брифы, лендинги).
- Изображения (PNG, JPG, WebP) для визуального анализа.
- Markdown-файлы.

## Какие отчёты формируются

- **Внутренний отчёт (internal mode)**: полный анализ с техническими деталями, таблица оценок, риски, рекомендации, контекст теста.
- **Клиентский отчёт (client-facing mode)**: адаптирован для показа клиенту, без внутренних формулировок и технических деталей.
- **HTML-отчёт**: стилизованная веб-версия для просмотра в браузере.
- **Markdown-отчёт**: для экспорта в документы.
- **A/B сравнение**: дельты оценок между вариантами, сводка сильных и слабых сторон.

## Как проходит демонстрационный сценарий

1. Загрузка трёх вариантов креатива для вымышленного банка NovaBank.
2. Запуск теста для каждого варианта.
3. Просмотр отчётов с оценками, рисками и рекомендациями.
4. Сравнение вариантов A/B/C.
5. Демонстрация обнаружения рискованных формулировок в варианте C.
6. Проверка closed-loop режима (все данные локальны, облачные сервисы не используются).

## Какие ограничения есть на MVP-этапе

- Используется SQLite (подходит для пилота, не для production-нагрузки).
- Симуляция аудитории — синтетическая (не заменяет real-user testing).
- Визуальный анализ — stub-режим (без GPU) или опционально локальный OCR/VLM.
- LLM-провайдер — stub (детерминированный) или опционально локальный (Ollama/LM Studio).
- Нет разграничения прав пользователей.
- Нет API-ключей или production-аутентификации.

## Что можно доработать после пилота

- Подключение реальной локальной LLM (Ollama, LM Studio) для более глубокого анализа.
- Подключение локального OCR (Tesseract) или VLM (Ollama vision model).
- Миграция на PostgreSQL для production-нагрузки.
- Интеграция с внешними системами через API.
- Добавление анализа видео.
- Расширение панели критериев под конкретные задачи клиента.

Система является **pre-test инструментом** для внутренней проверки креативов перед показом клиенту и не заменяет полноценные маркетинговые исследования.
