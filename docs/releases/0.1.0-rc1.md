# Creative Test Agent 0.1.0-rc1

## Status

Release candidate for controlled local pilot. Not yet production-ready. Intended for evaluation, demo, and structured pilot testing with a limited number of client campaigns.

## What is included

All core modules for creative pre-testing workflow:

- Client & Project workspace
- Creative asset management (text + file upload)
- Brand profile & audience profile management
- Brandbook upload, ingestion, and search
- Structured test runs with rubrics
- Batch campaign testing with A/B/C variants
- Report generation (JSON, Markdown, HTML)
- A/B comparison
- Report export (DOCX, PPTX, PDF-ready)
- Human-in-the-loop review workflow
- Knowledge base with keyword search
- Local job queue (SQLite-backed, no daemon)
- Guided demo flow
- Pilot readiness dashboard
- Release manifest & release check
- Client pilot pack builder
- Auth & role management (optional)
- Audit log & prompt tracing
- Evaluation framework for LLM quality control
- Server deployment profile (Docker Compose)
- Backup & restore

## Core workflows

1. **Demo**: Load NovaBank profile → Guided Demo → 10-step walkthrough
2. **Campaign testing**: Client → Project → Upload creatives → Upload brandbook → Create batch → Run → Summary → Export → Review
3. **Evaluation**: Run eval-stub → Review quality scores → Compare profiles
4. **Operations**: Backup → Restore → Maintenance → Export data

## Local-only guarantees

- All LLM inference runs locally (Ollama, LM Studio, or stub)
- No cloud LLM providers (OpenAI, Anthropic, Gemini, Perplexity)
- No external APIs
- No cloud storage
- No telemetry or external data transmission
- All data stored in local SQLite database

## How to run checks

```bash
make test              # pytest
make pilot-check       # integration checks
make check-server      # server readiness
make client-pilot-check  # client pack
make eval-stub         # evaluation
make release-manifest  # build manifest
make release-check     # full release check
make release-bundle    # build release artifact
make pilot-smoke       # smoke test
```

## Known limitations

- SQLite is not recommended for high-concurrency production workloads
- No real DOCX/PPTX generation — writes .md and .html files as placeholders
- No real PDF rendering — writes .html file as PDF-ready placeholder
- Vision analysis requires local VLM or stub mode
- No real-time video analysis
- No Redis/Celery/background worker — job queue is synchronous
- No Kubernetes or cloud orchestration
- No SSO/OAuth/enterprise auth — only local session auth

## Not included in this release

- Multi-tenant SaaS deployment
- Billing or subscription
- Cloud LLM integrations
- Real-time collaborative editing
- Mobile application
- WYSIWYG creative editor
- CI/CD pipelines
- Monitoring/alerting infrastructure

## Recommended pilot usage

1. Single-team evaluation (up to 5 users)
2. Up to 10 campaigns with 3-5 variants each
3. Local server or developer workstation
4. Stub mode for demonstrations
5. Docker Compose for server deployment
6. Regular backups via backup script
